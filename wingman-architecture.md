## 1. Product Overview

Wingman is a mobile-first PWA that acts as an AI dating message coach. A user installs it to their iPhone home screen, opens it while chatting on Hinge/Feeld/Bumble, snaps a screenshot of the conversation, selects a coaching stage (interest, attraction, rapport, desire, momentum, close), and receives 3 tailored message options generated by Claude.

**Core value loop:**
```
Screenshot → Stage selection → AI messages → Copy → Paste in dating app
```

**Monetization:** $19/month subscription via Stripe. Users get 3 free daily uses before hitting a paywall. Paid subscribers have unlimited usage.

## 5. Frontend — PWA

### 5.1 File: `public/index.html`

The entire frontend is a single HTML file with inline CSS and JS. No build step, no bundler. This is intentional — it deploys instantly, loads fast on mobile, and is trivial to iterate on.

**Screens (managed by JS `show(screenId)` function):**

| Screen ID | Purpose |
|---|---|
| `screen-splash` | App loading / auth check |
| `screen-auth` | Sign in / create account |
| `screen-home` | Main input: stage picker, screenshot upload, textarea, CTA |
| `screen-loading` | Animated loading while API call runs |
| `screen-results` | 3 message options as iMessage bubbles |
| `screen-paywall` | Upgrade prompt when free limit hit |
| `screen-account` | Subscription status, manage billing, sign out |

**Design spec:**
- Dark background: `#0a0a0f`
- Gold accent: `#c8a97e`
- Typography: Instrument Serif (headlines) + DM Sans (body)
- All screens respect iOS safe area insets (`env(safe-area-inset-top)`, `env(safe-area-inset-bottom)`)
- No user-scalable zoom (`maximum-scale=1.0, user-scalable=no` in viewport)
- `apple-mobile-web-app-capable` and `apple-mobile-web-app-status-bar-style: black-translucent` set

### 5.2 Firebase SDK loading (frontend)

Load Firebase from CDN in `index.html` — no npm, no bundler:

```html
<script type="module">
  import { initializeApp } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js';
  import { getAuth, GoogleAuthProvider, signInWithPopup,
           onAuthStateChanged, signOut } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-auth.js';
  import { getFirestore, doc, getDoc } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js';

  const app = initializeApp(firebaseConfig);
  const auth = getAuth(app);
  const db = getFirestore(app);

  // Auth state listener — runs on every page load
  onAuthStateChanged(auth, async (user) => {
    if (user) {
      await checkSubscriptionAndRoute(user);
    } else {
      show('screen-auth');
    }
  });
</script>
```

### 5.4 Making authenticated API calls

Every call to Cloud Functions includes a Firebase ID token in the `Authorization` header:

```javascript
async function callClaude(payload) {
  const token = await firebase.auth().currentUser.getIdToken();

  const res = await fetch('https://us-central1-wingman-app.cloudfunctions.net/claude', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify(payload)
  });

  if (res.status === 402) {
    show('screen-paywall');
    return null;
  }

  if (!res.ok) {
    const err = await res.json();
    throw new Error(err.message || 'Request failed');
  }

  return res.json();
}
```

### 5.5 Free usage tracking (frontend + Firestore)

Free users get **5 uses per day**. This is tracked in Firestore (server-authoritative), not localStorage. The frontend reads the count from the user doc on load and displays remaining uses.

The Cloud Function increments the count atomically and enforces the limit server-side. The frontend display is for UX only — the gate is always server-side.

### 5.7 File: `public/sw.js`

The service worker caches the app shell for fast repeat loads. It never caches API calls.

```javascript
const CACHE = 'wingman-v1';
const SHELL = ['/', '/index.html', '/manifest.json'];

self.addEventListener('install', e => {
  e.waitUntil(caches.open(CACHE).then(c => c.addAll(SHELL)).then(() => self.skipWaiting()));
});

self.addEventListener('activate', e => {
  e.waitUntil(
    caches.keys()
      .then(keys => Promise.all(keys.filter(k => k !== CACHE).map(k => caches.delete(k))))
      .then(() => self.clients.claim())
  );
});

self.addEventListener('fetch', e => {
  // Never cache: API calls, Stripe, Firebase, fonts
  const url = e.request.url;
  if (url.includes('anthropic.com') || url.includes('stripe.com') ||
      url.includes('firebaseapp.com') || url.includes('googleapis.com') ||
      url.includes('cloudfunctions.net')) return;

  e.respondWith(
    caches.match(e.request).then(cached => cached || fetch(e.request).then(res => {
      if (res.status === 200 && res.type === 'basic') {
        caches.open(CACHE).then(c => c.put(e.request, res.clone()));
      }
      return res;
    }))
  );
});
```

## 7. Stripe Integration

### 7.1 Stripe account setup

1. Create account at stripe.com
2. Create a **Product**: "Wingman Pro"
3. Create a **Price**: $19.00/month, recurring
4. Note the **Price ID** (looks like `price_1234abc...`) — this goes in environment config
5. Set up **Customer Portal** (stripe.com/docs/billing/subscriptions/customer-portal) — this lets users manage/cancel their own subscription without you building a UI

### 7.2 Checkout flow

When a user hits the paywall and taps "Upgrade", the frontend calls the `createCheckoutSession` Cloud Function, which creates a Stripe Checkout Session and returns a URL. The frontend redirects to that URL.

```
User taps "Upgrade"
  → POST /api/stripe/checkout (with Firebase auth token)
  → Cloud Function creates Stripe Checkout Session
  → Returns { url: "https://checkout.stripe.com/..." }
  → Frontend does window.location.href = url
  → User completes payment on Stripe's hosted page
  → Stripe redirects to https://wingman.app/?session_id={CHECKOUT_SESSION_ID}
  → Frontend detects ?session_id param, shows "Welcome to Pro" screen
  → Stripe webhook fires asynchronously, updates Firestore
```

### 7.3 Webhook events to handle

| Event | Action |
|---|---|
| `checkout.session.completed` | Write `subscription.status = 'active'`, store `stripeCustomerId` and `stripeSubscriptionId` |
| `invoice.paid` | Extend `currentPeriodEnd`, keep status `active` |
| `invoice.payment_failed` | Set `subscription.status = 'past_due'` |
| `customer.subscription.deleted` | Set `subscription.status = 'canceled'` |
| `customer.subscription.updated` | Sync any plan changes |

### 7.4 Customer portal

When a subscribed user taps "Manage Subscription" in `screen-account`, call `createPortalSession` Cloud Function which returns a Stripe Customer Portal URL. Redirect there. No custom cancellation UI needed.

## 8. Cloud Functions

All functions live in `functions/src/`. Export them all from `functions/src/index.ts`.

### 8.1 Function: `claude` (HTTPS)

**Path:** `POST /claude`  
**Auth required:** Yes (Firebase ID token)  
**Subscription required:** Paid OR free with remaining daily uses

```typescript
// functions/src/claude.ts
import * as functions from 'firebase-functions';
import * as admin from 'firebase-admin';
import fetch from 'node-fetch';
import { verifyAuth } from './middleware/auth';
import { checkUsage } from './middleware/subscription';
import { cors } from './lib/cors';

export const claude = functions
  .runWith({ timeoutSeconds: 60, memory: '512MB' })
  .https.onRequest(async (req, res) => {
    cors(req, res, async () => {
      if (req.method !== 'POST') return res.status(405).json({ error: 'Method not allowed' });

      // 1. Verify Firebase auth token
      const uid = await verifyAuth(req, res);
      if (!uid) return; // verifyAuth sends 401 response

      // 2. Check subscription / usage
      const allowed = await checkUsage(uid, res);
      if (!allowed) return; // checkUsage sends 402 response

      // 3. Validate request body
      const { messages } = req.body;
      if (!messages || !Array.isArray(messages)) {
        return res.status(400).json({ error: 'messages array required' });
      }

      // 4. Forward to Anthropic
      try {
        const anthropicRes = await fetch('https://api.anthropic.com/v1/messages', {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            'x-api-key': functions.config().anthropic.key,
            'anthropic-version': '2023-06-01',
          },
          body: JSON.stringify({
            model: 'claude-sonnet-4-20250514',
            max_tokens: 1000,
            messages,
          }),
        });

        const data = await anthropicRes.json();

        if (!anthropicRes.ok) {
          functions.logger.error('Anthropic error', data);
          return res.status(502).json({ error: 'AI service error', detail: data });
        }

        // 5. Increment usage count
        await incrementUsage(uid);

        return res.json(data);

      } catch (err) {
        functions.logger.error('Claude function error', err);
        return res.status(500).json({ error: 'Internal error' });
      }
    });
  });

async function incrementUsage(uid: string) {
  const today = new Date().toISOString().split('T')[0]; // 'YYYY-MM-DD'
  const ref = admin.firestore().collection('users').doc(uid);

  await admin.firestore().runTransaction(async (tx) => {
    const doc = await tx.get(ref);
    const data = doc.data()!;
    const usage = data.usage || {};

    if (usage.date === today) {
      tx.update(ref, { 'usage.count': admin.firestore.FieldValue.increment(1) });
    } else {
      tx.update(ref, { 'usage.date': today, 'usage.count': 1 });
    }
  });
}
```

### 8.2 Middleware: `auth.ts`

```typescript
// functions/src/middleware/auth.ts
import * as admin from 'firebase-admin';
import { Request, Response } from 'express';

export async function verifyAuth(req: Request, res: Response): Promise<string | null> {
  const header = req.headers.authorization;
  if (!header?.startsWith('Bearer ')) {
    res.status(401).json({ error: 'Missing auth token' });
    return null;
  }

  const token = header.split('Bearer ')[1];
  try {
    const decoded = await admin.auth().verifyIdToken(token);
    return decoded.uid;
  } catch (err) {
    res.status(401).json({ error: 'Invalid auth token' });
    return null;
  }
}
```

### 8.3 Middleware: `subscription.ts`

```typescript
// functions/src/middleware/subscription.ts
import * as admin from 'firebase-admin';
import { Response } from 'express';

const FREE_DAILY_LIMIT = 5;

export async function checkUsage(uid: string, res: Response): Promise<boolean> {
  const ref = admin.firestore().collection('users').doc(uid);
  const snap = await ref.get();

  if (!snap.exists) {
    res.status(403).json({ error: 'User not found' });
    return false;
  }

  const data = snap.data()!;
  const sub = data.subscription || {};

  // Paid subscribers: always allow
  if (sub.status === 'active') {
    // Also check subscription hasn't lapsed
    const periodEnd = sub.currentPeriodEnd?.toDate?.() || new Date(sub.currentPeriodEnd);
    if (periodEnd > new Date()) return true;
  }

  // Free users: check daily limit
  const today = new Date().toISOString().split('T')[0];
  const usage = data.usage || {};
  const todayCount = usage.date === today ? (usage.count || 0) : 0;

  if (todayCount >= FREE_DAILY_LIMIT) {
    res.status(402).json({
      error: 'Free limit reached',
      limit: FREE_DAILY_LIMIT,
      used: todayCount,
    });
    return false;
  }

  return true;
}
```

### 8.4 Function: `createCheckoutSession` (HTTPS)

```typescript
// functions/src/stripe.ts (partial)
export const createCheckoutSession = functions.https.onRequest(async (req, res) => {
  cors(req, res, async () => {
    const uid = await verifyAuth(req, res);
    if (!uid) return;

    const userDoc = await admin.firestore().collection('users').doc(uid).get();
    const userData = userDoc.data()!;

    // Create or retrieve Stripe customer
    let customerId = userData.subscription?.stripeCustomerId;
    if (!customerId) {
      const customer = await stripe.customers.create({
        email: userData.email,
        metadata: { firebaseUid: uid },
      });
      customerId = customer.id;
      await admin.firestore().collection('users').doc(uid).update({
        'subscription.stripeCustomerId': customerId,
      });
    }

    const session = await stripe.checkout.sessions.create({
      customer: customerId,
      payment_method_types: ['card'],
      mode: 'subscription',
      line_items: [{ price: functions.config().stripe.price_id, quantity: 1 }],
      success_url: `https://wingman.app/?session_id={CHECKOUT_SESSION_ID}`,
      cancel_url: `https://wingman.app/?checkout=canceled`,
      subscription_data: {
        metadata: { firebaseUid: uid },
      },
    });

    res.json({ url: session.url });
  });
});
```

### 8.5 Function: `stripeWebhook` (HTTPS)

This is the most critical function. Stripe calls it when subscription events occur. It must verify the webhook signature before processing.

```typescript
export const stripeWebhook = functions.https.onRequest(async (req, res) => {
  // NOTE: This function must receive the raw request body for signature verification.
  // In firebase.json, set "invoker": "public" and do NOT add CORS or body parsing.

  const sig = req.headers['stripe-signature'] as string;
  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(
      req.rawBody,  // Firebase Functions provides req.rawBody
      sig,
      functions.config().stripe.webhook_secret
    );
  } catch (err) {
    functions.logger.error('Webhook signature verification failed', err);
    return res.status(400).send('Webhook Error');
  }

  const db = admin.firestore();

  switch (event.type) {
    case 'checkout.session.completed': {
      const session = event.data.object as Stripe.Checkout.Session;
      const uid = session.subscription_data?.metadata?.firebaseUid
                  || await getUidByCustomerId(session.customer as string);
      if (!uid) break;

      await db.collection('users').doc(uid).update({
        'subscription.status': 'active',
        'subscription.stripeCustomerId': session.customer,
        'subscription.stripeSubscriptionId': session.subscription,
      });
      break;
    }

    case 'invoice.paid': {
      const invoice = event.data.object as Stripe.Invoice;
      const uid = await getUidByCustomerId(invoice.customer as string);
      if (!uid) break;

      const sub = await stripe.subscriptions.retrieve(invoice.subscription as string);
      await db.collection('users').doc(uid).update({
        'subscription.status': 'active',
        'subscription.currentPeriodEnd': new Date(sub.current_period_end * 1000),
      });
      break;
    }

    case 'invoice.payment_failed': {
      const invoice = event.data.object as Stripe.Invoice;
      const uid = await getUidByCustomerId(invoice.customer as string);
      if (!uid) break;
      await db.collection('users').doc(uid).update({ 'subscription.status': 'past_due' });
      break;
    }

    case 'customer.subscription.deleted': {
      const sub = event.data.object as Stripe.Subscription;
      const uid = await getUidByCustomerId(sub.customer as string);
      if (!uid) break;
      await db.collection('users').doc(uid).update({ 'subscription.status': 'canceled' });
      break;
    }
  }

  res.json({ received: true });
});

async function getUidByCustomerId(customerId: string): Promise<string | null> {
  const snap = await admin.firestore()
    .collection('users')
    .where('subscription.stripeCustomerId', '==', customerId)
    .limit(1)
    .get();
  return snap.empty ? null : snap.docs[0].id;
}
```

## 12. End-to-End User Flow

### New user, free tier

```
1. User opens https://wingman.app in Safari
2. Sees "Add to Home Screen" install prompt banner
3. Taps "Continue with Google" → Firebase Auth popup
4. Auth resolves → createUser Cloud Function fires → Firestore doc created
5. onAuthStateChanged fires → checkSubscriptionAndRoute() → show screen-home
6. User selects stage, uploads screenshot, taps "Get my messages"
7. Frontend calls GET /claude with Firebase ID token
8. Cloud Function: verifyAuth (OK) → checkUsage (count=0, limit=5, OK) → calls Anthropic
9. Anthropic returns 3 message options
10. Cloud Function: incrementUsage (count becomes 1) → returns response
11. Frontend renders results on screen-results
12. User taps bubble to copy → pastes in Hinge
```

### Free user hitting limit

```
1. User has used 5 free requests today
2. Taps "Get my messages"
3. Cloud Function: checkUsage → count=5 → returns HTTP 402
4. Frontend detects 402 → show screen-paywall
5. User taps "Upgrade — $19/mo"
6. Frontend calls /createCheckoutSession
7. Function creates Stripe session → returns checkout URL
8. Frontend redirects to Stripe Checkout
9. User enters card → completes payment
10. Stripe redirects to https://wingman.app/?session_id=...
11. Frontend shows "Welcome to Wingman Pro" screen
12. Async: Stripe fires checkout.session.completed webhook
13. stripeWebhook function updates Firestore: subscription.status = 'active'
14. Next API call: checkUsage sees 'active' → allows unlimited use
```

### Subscriber cancels

```
1. User taps "Manage Subscription" in screen-account
2. Frontend calls /createPortalSession
3. Function returns Stripe Customer Portal URL
4. Frontend redirects to portal
5. User cancels subscription
6. Stripe fires customer.subscription.deleted
7. stripeWebhook sets subscription.status = 'canceled'
8. Next API call: checkUsage sees 'canceled' → falls back to free tier limit
```

## 13. Security Rules

### `firestore.rules`

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only read/write their own document
    match /users/{userId} {
      allow read: if request.auth != null && request.auth.uid == userId;

      // Users cannot write directly — all writes go through Cloud Functions
      // (which use admin SDK and bypass these rules)
      allow write: if false;
    }

    // Deny everything else
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

**Key principle:** The client never writes to Firestore directly. All writes (subscription updates, usage counts) go through Cloud Functions that use the Firebase Admin SDK, which bypasses security rules entirely. This prevents clients from spoofing their own subscription status.

## 14. Cost Model

At $19/month per user, here's the unit economics:

| Service | Cost | Notes |
|---|---|---|
| Firebase Hosting | Free | Up to 10 GB/month |
| Firebase Auth | Free | Up to 10K users/month |
| Firestore | ~$0.01/user/month | ~30 reads + 30 writes/month per active user |
| Cloud Functions | ~$0.02/user/month | ~150 invocations/month per active user |
| Anthropic API | ~$2.50/user/month | ~150 calls/month, claude-sonnet-4-20250514 input+output |
| Stripe fees | ~$0.83/user/month | 2.9% + $0.30 per transaction |
| **Total cost** | **~$3.36/user/month** | |
| **Revenue** | **$19/user/month** | |
| **Gross margin** | **~82%** | |

At 50 subscribers: ~$775/month profit.  
At 200 subscribers: ~$3,100/month profit.

The main variable cost is Anthropic API usage. If users are very active (500+ calls/month), cost per user rises. Monitor via Anthropic dashboard and consider raising price or adding usage caps for power users if needed.
